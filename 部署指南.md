# 中医AI助手Web版 - Render部署指南

## 准备工作

### 1. 注册GitHub账号（如果没有）
访问 https://github.com 注册账号

### 2. 创建GitHub仓库

1. 登录GitHub
2. 点击右上角 "+" -> "New repository"
3. 仓库名：tcm-ai-assistant
4. 选择 Public（公开）
5. 点击 "Create repository"

### 3. 上传代码到GitHub

在项目目录下执行：

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/tcm-ai-assistant.git
git push -u origin main
```

## 部署到Render

### 1. 登录Render

访问 https://render.com 并登录你的账号

### 2. 创建新的Web Service

1. 点击 "New +" -> "Web Service"
2. 选择 "Connect a repository"
3. 授权GitHub访问
4. 选择 tcm-ai-assistant 仓库
5. 点击 "Connect"

### 3. 配置Web Service

Render会自动检测 `render.yaml` 配置文件，但你也可以手动配置：

- **Name**: tcm-ai-assistant（或自定义）
- **Environment**: Python 3
- **Build Command**: `pip install -r requirements.txt`
- **Start Command**: `gunicorn app:app`
- **Plan**: Free（免费方案）

### 4. 添加环境变量（可选）

如果需要修改配置，可以在 Environment 标签页添加：

- `PYTHON_VERSION`: 3.9.0
- `FREE_DAILY_LIMIT`: 5

### 5. 部署

点击 "Create Web Service"，Render会自动：

1. 克隆GitHub仓库
2. 安装依赖
3. 启动应用
4. 分配一个URL（如：https://tcm-ai-assistant.onrender.com）

部署过程大约需要3-5分钟。

### 6. 访问应用

部署完成后，点击分配的URL即可访问你的应用。

## 注意事项

### 免费方案限制

- 750小时/月运行时间
- 15分钟无活动后自动休眠
- 休眠后首次访问需要30秒左右唤醒
- 512MB内存
- 共享CPU

### 数据持久化

Render免费方案的文件系统是临时的，重启后会丢失。如果需要持久化数据：

1. 升级到付费方案使用Persistent Disk
2. 或使用外部数据库（如 ElephantSQL 的免费PostgreSQL）

### 自动部署

每次推送代码到GitHub main分支，Render会自动重新部署。

```bash
git add .
git commit -m "更新说明"
git push
```

## 自定义域名（可选）

1. 在Render控制台进入你的Web Service
2. 点击 "Settings" -> "Custom Domain"
3. 添加你的域名
4. 在域名DNS设置中添加CNAME记录

## 监控和日志

### 查看日志

在Render控制台：
1. 进入你的Web Service
2. 点击 "Logs" 标签
3. 实时查看应用日志

### 监控运行状态

在Dashboard可以看到：
- 运行时间
- 请求数量
- 响应时间
- 错误率

## 故障排查

### 应用无法启动

1. 检查 requirements.txt 是否正确
2. 查看 Logs 中的错误信息
3. 确认 gunicorn 已安装

### 数据库错误

1. 确认 data 目录存在
2. 检查数据库路径配置
3. 查看文件权限

### API调用失败

1. 检查通义千问API密钥是否正确
2. 确认API额度是否用完
3. 查看网络连接是否正常
